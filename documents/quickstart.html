Quick Start

Get OwnAudioSharp running in your .NET project in minutes.

🦀

New in 4.0: the entire audio path now runs in a purpose-built native Rust engine, wrapped in the same C# API you already use. The upgrade is fully backward compatible — your existing code keeps working, and gains a fully native real-time path. Read about the architecture →

Installation

Choose the package that fits your use case. Requires .NET 10.0 or later. The native Rust engine — including the built-in audio decoder — is bundled; there are no external codecs to configure.

PackagePlatformsWhat's included
OwnAudioSharpWindows, Linux, macOSFull desktop feature set — mixing, 17 effects, VST3, network sync, recording, mastering & analysis
OwnAudioSharp.MobileAndroid, iOSThe engine and API tuned for mobile real-time constraints
OwnAudioSharp.BasicAll platformsLightweight core audio, effects, VST3, sync, pitch/tempo — no AI/ML models
OwnAudioSharp.MidiWindows, Linux, macOSStandalone AOT-compatible MIDI library
OwnVocalRemover Add-onWindows, Linux, macOSOptional ONNX-based vocal separation, split out of the core API — reference only when needed
Shell
dotnet add package OwnAudioSharp          # Desktop — full features
dotnet add package OwnAudioSharp.Mobile   # Android / iOS
dotnet add package OwnAudioSharp.Basic    # Lightweight
dotnet add package OwnVocalRemover        # Optional vocal separation add-on
🎵

Formats out of the box: the pure-Rust decoder handles MP3, FLAC, WAV (PCM/ADPCM), AAC, MP4/M4A, OGG/Vorbis and AIFF on every platform — no FFmpeg, no system dependencies. FFmpeg 8+ remains supported as an optional legacy fallback for exotic formats; see FFmpeg Integration.

Architecture

OwnAudioSharp is a thin C# surface over a native Rust engine. The design keeps the real-time audio thread — now entirely inside Rust — completely separate from your application code and the managed runtime.

Thread diagram
UI / Main Thread
  └─> OwnaudioNet.Send()     [lock-free ring buffer, <0.1ms]
       └─> AudioEngineWrapper   [managed bridge, non-blocking]
            └─> Native Rust engine   [ownaudio-ffi + ownaudio-core]
                 └─> Audio RT Thread  [WASAPI / CoreAudio / ALSA — fully native]
⚠️

Never call blocking engine methods from the UI thread. Use InitializeAsync(), StopAsync(), and ShutdownAsync() in all UI applications (WPF, WinForms, MAUI, Avalonia).

Basic Playback

The minimal setup to play an audio file:

C#
using OwnaudioNET;
using OwnaudioNET.Mixing;
using OwnaudioNET.Sources;

// 1. Initialize (async is required in UI apps)
await OwnaudioNet.InitializeAsync();
OwnaudioNet.Start();

// 2. Create the mixer
var mixer = new AudioMixer(OwnaudioNet.Engine!.UnderlyingEngine);
mixer.Start();

// 3. Load and play
var track = new FileSource("song.mp3");
mixer.AddSource(track);

// 4. Cleanup on app exit
mixer.Dispose();
await OwnaudioNet.ShutdownAsync();

Custom Configuration

C#
// Select WASAPI on Windows, use low latency
var config = new AudioConfig
{
    SampleRate = 48000,
    Channels   = 2,
    BufferSize = 256,   // ~5.3ms latency
    HostType   = OperatingSystem.IsWindows()
                    ? EngineHostType.WASAPI
                    : EngineHostType.None,
    EnableInput  = false,
    EnableOutput = true
};

await OwnaudioNet.InitializeAsync(config);
OwnaudioNet.Start();

Multi-Track with Sync

C#
var mixer = new AudioMixer(OwnaudioNet.Engine!.UnderlyingEngine, bufferSizeInFrames: 1024);
mixer.Start();

// Load tracks
var sr = OwnaudioNet.Engine!.Config.SampleRate;
var ch = OwnaudioNet.Engine!.Config.Channels;

var vocals  = new FileSource("vocals.wav",  targetSampleRate: sr, targetChannels: ch);
var backing = new FileSource("backing.mp3", targetSampleRate: sr, targetChannels: ch);

// Attach to master clock for sample-accurate sync
vocals.AttachToClock(mixer.MasterClock);
backing.AttachToClock(mixer.MasterClock);

// Set volume and stereo pan
vocals.Volume  = 0.9f;
backing.Volume = 0.8f;
vocals.Pan  = -0.2f;   // slightly left
backing.Pan =  0.2f;   // slightly right

// Seek and play
vocals.Seek(0);
backing.Seek(0);
vocals.Play();
backing.Play();

mixer.AddSource(vocals);
mixer.AddSource(backing);

Add Effects

C#
using OwnaudioNET.Effects;
using OwnaudioNET.Sources;

// Per-track effect
var source = new FileSource("guitar.wav");
var trackFx = new SourceWithEffects(source);
trackFx.AddEffect(new ReverbEffect { RoomSize = 0.6f, Mix = 0.25f });
trackFx.AddEffect(new CompressorEffect { Ratio = 4f, AttackTimeMs = 10f });
mixer.AddSource(trackFx);

// Master bus effect
mixer.AddMasterEffect(new LimiterEffect());
mixer.AddMasterEffect(new EqualizerEffect());

VST3 Plugin

C#
using OwnaudioNET.Effects;

var host = await VST3PluginHost.CreateAsync("/path/to/reverb.vst3");
int sr = OwnaudioNet.Engine!.Config.SampleRate;
await host.InitializeAudioAsync(sr, maxBlockSize: 1024);

// Use as master effect
mixer.AddMasterEffect(host.GetProcessor());
